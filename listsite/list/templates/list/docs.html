Introduction

list is a micro REST API to manage and view simple lists, written in Django. list is publicly accessible, and uses no security procedures. It stores data in a sqlite3 database.
All requests are POST requests, and must include the header {"content-type": "application/json"} On error, all endpoints will respond with status code 200, but the response will include an "error" field.

/newlist

Creates a new list with a random 6-character id on POST request. Request Parameters: none Response Parameters:
  • "list_id": id of the new list

/validatelist

Validates an existance of a list given its id Request Parameters:
  • "list_id": id of the list to validate
Response Parameters:
  • "exists": True if the list exists, false if not

/viewlist

Returns the items in the list given the id Request Parameters:
  • "list_id": id of the new list
Response Parameters:
  • "list_items": List of item objects, which have the form
    {"id": item id, "item": item text, "list": list id}

/additem

Adds an item to the given list Request Parameters:
  • "list_id": id of the list
Response Parameters:
  • "item": item which was added in the form {"id": item id, "item": item text, "list": list id}

/deleteitem

Deletes an item from the given list Request Parameters:
  • "list_id": id of the list
  • "item": item text of item to delete
Response Parameters: none

/clearlist

Clears all items from a list Request Parameters:
  • "list_id": id of the list
Response Parameters: none
